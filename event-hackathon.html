Hackathon Rules

  • Application Developent: In this phase the team need to develop an application for a given problem statement based on the topic they selected within 6/7 hours on spot in the event day. The problem statement will be given on spot in the event day.
  • Presentation: After the application development phase, each team needs to give a presentation based on the prototype or application they developed.
  • Basic Rules: The application must be developed on any of the following platforms:
    • * Web
    • * iOS
    • * Android
    • * Desktop
    • * The problem statement and challenges will be given on spot in the event day.
    • * The developed application does not need to be 100% complete. Building only the prototype and how the prototype is related with the problem statement is enough.
    • * Each team can have maximum 3 members. The members must be from SIU.
    • * Each team must have a team leader.
    • * The team members need to bring their own laptops, cell phones, mobile charger, power bank, multiplug etc. It is better if the team keeps an USB internet modem with them so that they can use it if any unavoidable circumstances occurs.
    • * Only registered teams will be allowed to participate in the hackathon.
    • * Teams must be present on time in the event day.
    • * The organizing committee holds the right to disqualify any team. Any type of misbehavior with the judges will result in disqualification.
    • * The organizing committee will not be liable if any team or individual loses his/her laptop, cellular phones, gadgets and other stuff.
